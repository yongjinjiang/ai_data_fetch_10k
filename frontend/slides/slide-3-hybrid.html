Slide 3 · Hybrid (Rule + Row-Candidate LLM)

Algorithm

  • Generate candidate rows per field from tables.
  • Send only candidate JSON to LLM for picking best value.
  • Validator arbitrates by confidence + sanity checks.

Key Result

Exact Accuracy: 73.3% (22/30) · MAPE: 4.65%

Why underperformed: LLM did not see full table structure (column/year/unit context), so some semantic mismatches remained.

Takeaway

LLM helps, but context granularity matters: row-only context is often insufficient for financial tables.
