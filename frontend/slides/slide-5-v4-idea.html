Slide 5 · V4 Design Thought (Human-like Table Reading)

Core Design Principle

A more robust approach is to have the LLM focus on what it does best: semantic understanding of table structure. For each table, the model identifies the most likely row and column headers for the target field and returns a confidence score. Python then deterministically reads the value from the located cell. This design aligns with human review behavior and handles transposed layouts, reordered headers, and non-pivot table structures more reliably.

Architecture Shift

  • LLM = semantic locator (table_id, row_label, column_label, confidence)
  • Python = deterministic reader (cell parsing, sign/unit normalization)
  • Reduce hallucination risk by removing direct LLM numeric output.
  • Better fit for transposed/reordered/non-pivot tables.
