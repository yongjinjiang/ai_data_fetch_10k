Slide 5 · V4 Design Thought (Human-like Table Reading)

Core Thought (CN)

“我想更好的方法可能是，让LLM针对每个表格的行和列名，进行智能判断，它是不是我们需要的field， 然后返回可信度最高的行列指标，直接由python去读结果……ai只需要专注于行和列（行列可能交换， 还可能是重排成长表格的非透视结构）两个名字，自然语言是它最擅长的。”

Architecture Shift

  • LLM = semantic locator (table_id, row_label, column_label, confidence)
  • Python = deterministic reader (cell parsing, sign/unit normalization)
  • Reduce hallucination risk by removing direct LLM numeric output.
  • Better fit for transposed/reordered/non-pivot tables.
