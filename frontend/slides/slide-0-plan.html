Slide 4.1 · My Next-Step Thought Process

Strategic Hypothesis

To further improve the extraction pipeline, I intentionally introduced additional, more challenging target fields. I expected accuracy to decline at first; that decline would surface hidden failure modes and guide the next round of targeted improvements.

Why this is a strong strategy

  • Don’t optimize for a static benchmark only — increase task difficulty on purpose.
  • Use performance drop as a diagnostic signal to expose hidden failure modes.
  • Apply root-cause analysis, then iterate with targeted fixes.
  • Demonstrates engineering maturity: stress test → analyze → improve.

Execution Loop

  1. Add one challenging field.
  2. Re-run all existing extractors (no architecture change at first).
  3. Inspect failures by company/statement/table pattern.
  4. Introduce focused strategy upgrades and re-evaluate.
