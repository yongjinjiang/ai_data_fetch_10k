10-K Financial Data Extraction: AI-Assisted Optimization Journey

Interview project focus: show how extraction quality improves step-by-step using better rules, hybrid strategies, and table-centric LLM reasoning.

Project Goal

Extract three fields from SEC 10-K filings for 10 companies: total_revenue, net_income, total_assets. Then evaluate accuracy against manually curated ground truth and iteratively improve.

Why this project matters

  • Demonstrates practical AI engineering (not just one-shot prompting).
  • Shows robust pipeline design: data ingestion → extraction → evaluation → error analysis → iteration.
  • Highlights tradeoffs between deterministic rules and LLM flexibility.

How to run each solution

# Setup
./venv/bin/pip install -r requirements.txt

# Baseline
./venv/bin/python main.py --skip-download --version v1

# Improved rules
./venv/bin/python main.py --skip-download --version v2

# Existing hybrid (rule + row-candidate LLM)
./venv/bin/python main.py --skip-download --version hybrid

# New table-centric solution
./venv/bin/python main.py --skip-download --version v3
